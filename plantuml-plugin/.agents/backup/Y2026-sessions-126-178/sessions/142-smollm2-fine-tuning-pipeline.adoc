== Problèmes rencontrés

1. **GGUF inutilisable pour fine-tuning** : le blob Ollama local (144 Mo) est format GGUF, utilisable uniquement en inférence. Le fine-tuning nécessite les poids HuggingFace (safetensors, ~270 Mo).
2. **pip install dans testcontainer trop lent** : `pip install torch transformers peft trl` prend 8-10 min en CPU → timeout des tests. Résolu avec une image Docker pré-buildée (`plantuml-fine-tune:latest`).
3. **Path en dur du venv local** : première version de `downloadFineTuningModel` utilisait `/home/cheroliv/workspace/runtimes/venv/bin/python3`. Corrigé : tout passe par `docker run`.
4. **@EnabledIfSystemProperty vs -P Gradle** : les propriétés Gradle (`-P`) ne sont pas des propriétés système Java (`-D`). Corrigé en forwardant `test.tags` dans `systemProperty()` du block `functionalTest {}`.
5. **NPE avec ToStringConsumer + followOutput** : conflit entre `withLogConsumer(ToStringConsumer())` et `followOutput()`. Simplifié en lambda `withLogConsumer { frame -> log.debug(...) }`.
